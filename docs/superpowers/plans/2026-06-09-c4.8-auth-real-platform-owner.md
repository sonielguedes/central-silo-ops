# C4.8 Auth Real + Platform Owner Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace mock/local login with real production-grade authentication using hashed passwords, signed sessions, and a seeded platform owner plus demo tenant.

**Architecture:** Introduce a server-side auth store backed by JSON files for users and sessions. Login issues an opaque signed session cookie; all protected APIs resolve identity from that session, then derive compatibility headers for legacy RBAC paths. Keep tenant isolation unchanged and layer auth on top without reworking operational storage.

**Tech Stack:** Next.js App Router, Node `crypto`, bcrypt-compatible hashing, JSON file storage, existing RBAC helpers, React context, fetch with `credentials: 'include'`.

  if (!cookie) return null;
  return AuthStore.resolveSession(cookie);
}
